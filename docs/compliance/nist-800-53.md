# NIST 800-53 Rev 5 Compliance Mapping

This document maps Mantissa Stance policy IDs to NIST Special Publication 800-53 Revision 5 security and privacy controls.

## Overview

- **Framework**: NIST SP 800-53 Rev 5
- **Title**: Security and Privacy Controls for Information Systems and Organizations
- **Total Control Families**: 20
- **Total Controls**: 1000+
- **Covered by Stance**: Access Control, Identification and Authentication, System and Communications Protection

## Policy Details by Control

### AC-6: Least Privilege

#### gcp-iam-003: Overly Permissive IAM Bindings
- **NIST Control**: AC-6
- **Control Enhancement**: Base control
- **Evidence Collection**:
  1. Run Stance scan to identify overly permissive bindings
  2. Flag allUsers and allAuthenticatedUsers members
  3. Document excessive permissions
  4. Export IAM policy analysis

#### gcp-iam-004: Service Account Admin Privileges
- **NIST Control**: AC-6(5)
- **Control Enhancement**: Privileged Accounts
- **Evidence Collection**:
  1. Run Stance scan to identify privileged service accounts
  2. Document admin role assignments
  3. Verify necessity of privileges
  4. Export service account permissions

#### azure-identity-001: Privileged Role Review
- **NIST Control**: AC-6(7)
- **Control Enhancement**: Review of User Privileges
- **Evidence Collection**:
  1. Run Stance scan to identify privileged assignments
  2. Document Owner, Contributor roles at subscription scope
  3. Verify access review process
  4. Export role assignment inventory

### IA-2: Identification and Authentication

#### aws-iam-001: Root Account MFA
- **NIST Control**: IA-2(1), IA-2(2)
- **Control Enhancement**: Multi-factor Authentication
- **Evidence Collection**:
  1. Run Stance scan to verify MFA configuration
  2. Document MFA device types
  3. Verify MFA enforcement policies
  4. Export MFA status report

#### aws-iam-002: Password Policy
- **NIST Control**: IA-5(1)
- **Control Enhancement**: Password-based Authentication
- **Evidence Collection**:
  1. Run Stance scan to verify password policy
  2. Document complexity requirements
  3. Verify password history settings
  4. Export password policy configuration

### SC-7: Boundary Protection

#### Network Security Policies
- **NIST Control**: SC-7
- **Control Enhancement**: Base control
- **Evidence Collection**:
  1. Run Stance scan to collect firewall rules
  2. Document network segmentation
  3. Identify unrestricted access
  4. Export network boundary configuration

### SC-28: Protection of Information at Rest

#### Encryption Policies
- **NIST Control**: SC-28(1)
- **Control Enhancement**: Cryptographic Protection
- **Evidence Collection**:
  1. Run Stance scan to verify encryption at rest
  2. Document encryption algorithms
  3. Verify key management practices
  4. Export encryption configuration

## Gaps and Recommendations

### Critical Gaps by Control Family

1. **AU - Audit**: Need CloudTrail/Audit Log validation
2. **CP - Contingency**: No backup/recovery validation
3. **IR - Incident Response**: No incident detection integration
4. **PE - Physical**: Not applicable to cloud-only assessment

### High-Priority Controls to Add

1. AU-3: Content of Audit Records
2. CM-2: Baseline Configuration
3. CP-9: System Backup
4. SC-5: Denial of Service Protection
5. SI-3: Malware Protection

## Running NIST 800-53 Compliance Checks

```bash
# Scan for NIST 800-53 compliance
stance scan --framework nist-800-53

# Generate NIST compliance report
stance report --framework nist-800-53 --format html

# Query by control family
stance query "show AC family findings"
stance query "show SC-28 encryption compliance"

# Export for POAM development
stance findings --framework nist-800-53 --format csv --output nist-findings.csv
```

## System Security Plan (SSP) Support

Stance findings can support SSP development:

1. **Control Implementation Statements**: Document how controls are met
2. **Evidence Artifacts**: Export scan results as evidence
3. **Continuous Monitoring**: Track control effectiveness over time
4. **POAM Development**: Identify gaps requiring remediation
